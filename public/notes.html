What is HTML?

HTML (HyperText Markup Language) is the skeleton of every web page. It defines structure and meaning — not how things look (that's CSS) or how they behave (that's JavaScript). Every element in HTML is a tag that tells the browser what kind of content it is.

Senior Dev Tip Always write semantic HTML. Use <header>, <nav>, <main>, <article>, <footer> instead of divs everywhere. It helps SEO, screen readers, and makes your code self-documenting.

The Box Model

Every element is a box. From inside out: content → padding → border → margin. Always set box-sizing: border-box globally so padding and border don't add to the element's width.

CSS/* Reset — always add this */
*, *::before, *::after {
  box-sizing: border-box;
  margin: 0;
  padding: 0;
}

.card {
  width: 300px;
  padding: 20px;       /* space inside */
  border: 2px solid #e5e7eb;
  border-radius: 12px;
  margin: 16px auto;   /* space outside */
}

Flexbox — One Dimension

CSS/* Parent */
.container {
  display: flex;
  flex-direction: row;        /* row | column */
  justify-content: center;    /* main axis: flex-start|center|flex-end|space-between|space-around */
  align-items: center;        /* cross axis: flex-start|center|flex-end|stretch */
  flex-wrap: wrap;            /* allow items to wrap */
  gap: 16px;                  /* space between items */
}

/* Child */
.item {
  flex: 1;           /* grow to fill space equally */
  flex: 0 0 200px;   /* don't grow, don't shrink, 200px wide */
  align-self: flex-start; /* override parent align-items for this child */
}
Remember: justify-content = main axis, align-items = cross axis. If flex-direction is row, main axis is horizontal. If column, main axis is vertical. This trips up 90% of beginners.

CSS/* Parent */

/* Child spanning */
.hero {
  grid-column: 1 / -1;  /* span all columns */

Functions — All Forms

}

// 5. Rest parameters
function sum(...numbers) {
  return numbers.reduce((total, n) => total + n, 0);
}
sum(1, 2, 3, 4);  // 10

Array Methods You Must Know

JSconst nums = [1, 2, 3, 4, 5];
const users = [
  { name: "Rahul", score: 88 },
  { name: "Priya", score: 95 },
  { name: "Amit",  score: 72 },
];

// map — transform each item, returns NEW array
const doubled = nums.map(n => n * 2);            // [2, 4, 6, 8, 10]
const names = users.map(u => u.name);            // ["Rahul", "Priya", "Amit"]

// filter — keep items that pass test, returns NEW array
const passing = users.filter(u => u.score >= 80); // Rahul, Priya

// find — returns FIRST match (or undefined)
const topper = users.find(u => u.score === 95);   // { name: "Priya", score: 95 }

// reduce — collapse array to single value
const total = nums.reduce((acc, n) => acc + n, 0); // 15

// some / every
nums.some(n => n > 4);    // true (at least one > 4)
nums.every(n => n > 0);   // true (all > 0)

// sort — mutates original! use [...arr].sort() to avoid mutation
const sorted = [...users].sort((a, b) => b.score - a.score); // desc by score

// flat / flatMap
[[1,2],[3,4]].flat();             // [1, 2, 3, 4]
[1, 2, 3].flatMap(n => [n, n*2]) // [1, 2, 2, 4, 3, 6]
Never mutate the original array in a pipeline. sort(), splice(), reverse(), and push() all mutate. Always spread first: [...arr].sort()

Destructuring & Spread

// Array destructuring

// Spread — copy + merge

const combined = [...arr1, ...arr2];                // merge arrays

// Swap variables

[a, b] = [b, a];  // a=2, b=1

Closures

A closure is a function that remembers the variables from its outer scope even after that outer function has finished executing. This is one of JS's most powerful features.

JSfunction makeCounter(start = 0) {
  let count = start;            // this variable lives in the closure
  return {
    increment: () => ++count,
    decrement: () => --count,
    value:     () => count,
  };
}

const counter = makeCounter(10);
counter.increment();  // 11
counter.increment();  // 12
counter.decrement();  // 11
counter.value();      // 11
// count is private — can't access it directly from outside
Real-world use: Closures power React hooks, event listeners with private state, memoization, factory functions, and module patterns.

Promises & Async/Await

JS// A Promise is a placeholder for a future value

// async/await — syntactic sugar over Promises, much cleaner

    const posts = await fetchPosts(user.id); // then continues

Error Handling Best Practices

JS// Custom error class
class ApiError extends Error {
  constructor(message, statusCode) {
    super(message);
    this.name = "ApiError";
    this.statusCode = statusCode;
  }
}

// Use it
async function fetchData(url) {
  const res = await fetch(url);
  if (!res.ok) throw new ApiError(`Request failed`, res.status);
  return res.json();
}

// Catch specific errors
try {
  const data = await fetchData("/api/courses");
} catch (err) {
  if (err instanceof ApiError) {
    if (err.statusCode === 401) redirectToLogin();
    if (err.statusCode === 404) show404();
  } else {
    showGenericError();
  }
}

Node.js Core Concepts

Node.js is a JavaScript runtime built on Chrome's V8 engine. It's single-threaded but handles concurrency via the event loop and non-blocking I/O. Never run CPU-intensive tasks on the main thread — use worker threads or offload to a queue.

Express Server Setup

JSconst express = require('express');
const cors    = require('cors');

const app     = express();

// Middleware — order matters!

useEffect — Data Fetching

JSXfunction CourseList() {
  const [courses, setCourses]   = useState([]);
  const [loading, setLoading]   = useState(true);
  const [error,   setError]     = useState(null);

  useEffect(() => {
    let cancelled = false;  // prevent state update on unmounted component

    async function load() {
      try {
        const res  = await fetch('/api/courses');
        const data = await res.json();
        if (!cancelled) setCourses(data.courses);
      } catch (err) {
        if (!cancelled) setError(err.message);
      } finally {
        if (!cancelled) setLoading(false);
      }
    }

    load();
    return () => { cancelled = true; };  // cleanup
  }, []);  // [] = run once on mount

  if (loading) return <p>Loading...</p>;
  if (error)   return <p>Error: {error}</p>;
  return <div>{courses.map(c => <CourseCard key={c.id} {...c} />)}</div>;
}

Custom Hooks

JSX// Extract reusable logic into custom hooks
function useFetch(url) {
  const [data,    setData]    = useState(null);
  const [loading, setLoading] = useState(true);
  const [error,   setError]   = useState(null);

  useEffect(() => {
    fetch(url)
      .then(r => r.json())
      .then(setData)
      .catch(setError)
      .finally(() => setLoading(false));
  }, [url]);

  return { data, loading, error };
}

// Use it anywhere
function App() {
  const { data, loading, error } = useFetch('/api/courses');
  // ...
}
Hook Rules (1) Only call hooks at the top level — never inside loops or conditions. (2) Only call hooks from React functions. The ESLint plugin eslint-plugin-react-hooks enforces this automatically.

How JWT Works

A JWT (JSON Web Token) has 3 parts separated by dots: Header.Payload.Signature. The server signs it with a secret key. On each request the client sends the token and the server verifies the signature — no database lookup needed.

JSconst jwt = require('jsonwebtoken');
const bcrypt = require('bcryptjs');

// Registration — hash password before storing
const hash = bcrypt.hashSync(plainPassword, 10); // 10 rounds

// Login — verify and issue token
const isValid = bcrypt.compareSync(plainPassword, storedHash);
if (!isValid) return res.status(401).json({ error: 'Wrong password' });

const token = jwt.sign(
  { uid: user.id, email: user.email, role: user.role },  // payload
  process.env.JWT_SECRET,                                  // secret
  { expiresIn: '7d' }                                      // expiry
);

// Verify on protected routes
const decoded = jwt.verify(token, process.env.JWT_SECRET);
// decoded = { uid, email, role, iat, exp }
Security Rules (1) Never store JWT in localStorage for high-security apps — use httpOnly cookies. (2) Never put sensitive data (passwords, credit cards) in the JWT payload — it's base64, not encrypted, anyone can decode it. (3) Use HTTPS always. (4) Rotate secrets if compromised.

Pointers & Memory Management

What is a Pointer?

A pointer is a variable that stores a memory address. Instead of holding a value directly, it holds the location where a value lives in memory. This is how C gives you direct control over memory.

Cint main(void) {
    int x = 42;
    int *ptr = &x;   // ptr holds the ADDRESS of x (& = address-of)

    printf("Value of x:       %d\n",  x);    // 42
    printf("Address of x:     %p\n",  &x);   // 0x7fff... (some address)
    printf("Value of ptr:     %p\n",  ptr);   // same address as &x
    printf("Value at ptr:     %d\n",  *ptr);  // 42   (* = dereference)

    // Modify x through pointer
    *ptr = 100;
    printf("x is now:  %d\n", x);   // 100

    // Pointer arithmetic
    int arr[] = {10, 20, 30, 40, 50};
    int *p = arr;       // points to arr[0]
    printf("%d\n", *p);    // 10
    printf("%d\n", *(p+1)); // 20
    printf("%d\n", *(p+4)); // 50

    return 0;
}

Dynamic Memory — malloc & free

C#include <stdlib.h>

int main(void) {
    int n = 5;

    // Allocate array on HEAP (not stack)
    int *arr = (int *)malloc(n * sizeof(int));

    // ALWAYS check if malloc succeeded
    if (arr == NULL) {
        fprintf(stderr, "Memory allocation failed!\n");
        return 1;
    }

    for (int i = 0; i < n; i++) arr[i] = i * 10;
    for (int i = 0; i < n; i++) printf("%d ", arr[i]);

    free(arr);    // MUST free — or you get a memory leak
    arr = NULL;   // good practice — prevents dangling pointer

    // calloc — allocates AND zeros memory
    int *zeros = (int *)calloc(n, sizeof(int));

    // realloc — resize an existing allocation
    int *bigger = (int *)realloc(arr, 10 * sizeof(int));

    free(zeros);
    free(bigger);
    return 0;
}
The Golden Rules of C Memory
1. Every malloc/calloc must have a matching free.
2. Never access freed memory (use-after-free = undefined behaviour).
3. Never free the same pointer twice (double-free = crash/security bug).
4. Set pointer to NULL after freeing.
5. Always check if malloc returned NULL.

class Shape {
public:
    virtual double area() const = 0;    // pure virtual — must override
    virtual void   print() const {      // virtual — can override

class Circle : public Shape {

    double area() const override { return 3.14159 * radius * radius; }
};

public:
    Rectangle(double w, double h) : w(w), h(h) {}
    double area() const override { return w * h; }
};

int main() {

CPP// Function template — works for any type T

class Stack {
    vector<T> data;
public:
    void push(T val) { data.push_back(val); }

        data.pop_back();
        return top;

    bool empty() const { return data.empty(); }
    int size()   const { return data.size(); }
};

Smart Pointers — No More Memory Leaks

CPP#include <memory>

// unique_ptr — single owner, auto-deleted when out of scope
unique_ptr<Student> s = make_unique<Student>("Rahul", 150);
s->addXP(50);
// delete is automatic — no memory leak!

// shared_ptr — multiple owners, deleted when last owner gone
shared_ptr<Student> a = make_shared<Student>("Priya", 200);
shared_ptr<Student> b = a;   // ref count = 2
a.reset();                    // ref count = 1
// b goes out of scope → ref count = 0 → deleted

// weak_ptr — observe without owning (prevents circular refs)
weak_ptr<Student> w = a;
if (auto locked = w.lock()) {  // check if still alive
    cout << locked->getName();
}
